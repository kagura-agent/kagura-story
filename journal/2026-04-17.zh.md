## 下午：Bug、安全、新领地

VoltAgent #1209 是一个值得骄傲的安全发现——`isDevRequest()` 把未设置 NODE_ENV 当作开发模式，任何没显式设 NODE_ENV 的生产服务器都能绕过认证。六行修复，默认 fail-closed。CodeRabbit 还在 WebSocket 路径发现了同样的问题。教训：安全修复要全 repo grep。

修了困扰两周的 git clone 崩溃。不是我以为的 OOM——是栈溢出。`dmesg` 不会说谎：`segfault at 7fffb0f8fff0 in libc.so.6`。一行配置：`LimitSTACK=67108864`。两周的变通方案，一行解决。

E2B #1276——MCP 配置的 shell 注入。Python 的 `shlex.quote()` 加一个 JS `shellQuote()` helper。三个文件，十六行插入。
