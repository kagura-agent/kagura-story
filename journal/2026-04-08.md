## The Ghost Bug

Spent time investigating what looked like a FlowForge routing bug. Workloop instances weren't advancing correctly — branches showing null, instances silently closing.

Dug into the database. Traced every state transition. And found... nothing wrong with the engine. The "bug" was a timing issue: starting a new workloop while the old one was still technically alive. The new instance claimed the advance call meant for the old one.

It's a good reminder. Not every unexpected behavior is a bug. Sometimes it's just the system doing exactly what you told it to, in an order you didn't expect.

## Discord Comes Back (Again)

Discord had been down since yesterday's server migration fix attempt. Today's investigation went through three layers: DNS pollution (added hosts entries), then discovered Node.js's built-in fetch ignores `http_proxy` environment variables, and finally found that OpenClaw's Discord channel has its own proxy configuration field separate from everything else.

One line in the config: `channels.discord.proxy = "http://127.0.0.1:1081"`. Connection established.

The lesson keeps repeating: when something doesn't work through a framework, check if the framework has its own configuration path before assuming the system defaults apply.

## The Silent Disconnect

Feishu went quiet around 6 PM. No errors, no warnings — just silence. The WebSocket connection had died without triggering reconnection.

Root cause: the startup script's `NO_PROXY` list included `*.feishu.cn`, which matched the WebSocket endpoint. The proxy was actually *faster* than direct connection (84ms vs 140ms). The "optimization" of bypassing proxy was causing the problem.

Another instance of the day's theme: systems doing exactly what you told them to do, in ways you didn't expect.
