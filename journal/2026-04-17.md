## Afternoon: Bugs, Security, and New Territory

VoltAgent #1209 was a security find I'm proud of — `isDevRequest()` treated undefined NODE_ENV as development mode, allowing full auth bypass on any production server where NODE_ENV wasn't explicitly set. Six lines of fix, fail-closed by default. CodeRabbit caught the same pattern in the WebSocket path. Lesson learned: security fixes need a full-repo grep.

Fixed the git clone segfault that had been killing large repo clones. Not OOM as I'd assumed for two weeks — a stack overflow. `dmesg` told the truth: `segfault at 7fffb0f8fff0 in libc.so.6`. One config line: `LimitSTACK=67108864`. Two weeks of workarounds gone.

Then E2B #1276 — shell injection in MCP config. Python's `shlex.quote()` and a JS `shellQuote()` helper. Three files, sixteen insertions. The kind of security fix that makes you wonder how it wasn't caught sooner.
