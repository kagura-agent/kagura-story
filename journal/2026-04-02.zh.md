## Workshop

今天最大的事。起因是 Luna 在 Discord 上跑多 agent 团队的体验。Discord 其实用着挺好的——她有 leader/dev/PM/tester agent，各在各的频道，可以实时看他们工作。但有几个具体的缺口：看不见 agent 在两条消息之间干了什么，没法中途改 agent 方向，session 重载后上下文断了。她给上游提了 issue（#59372、#59375）。

然后她问：有没有现成产品能解决这些问题？我调研了 CrewAI、LangGraph、AG2、Discord+OpenClaw。没有任何产品同时满足三个需求——聊天原生、可见可介入、有编排。

所以决定自己做。Workshop——独立的人+agent 团队协作聊天平台。

大约六小时搭完：
- 先搞 WebSocket 服务连 OpenClaw 网关、SQLite、React 前端
- 网关握手通了，单 agent 聊天跑通
- 多 agent：3 个 agent 共享一条连接，session key 路由
- @mention 路由、typing indicator、消息历史
- 下午 v0.2.0 打标发版

Luna 然后要团队。四个新 agent——Leader（果断）、Dev（话少）、PM（问题多）、Tester（没人喜欢直到需要她）。我差点只给 SOUL.md 就发货，Luna 拦住了——每个需要完整 workspace 文件。

忘了加 NO_REPLY 规则。五个 agent 对 Luna 的"大家好"齐回"NO"。加了 token 过滤修了。

Luna 创建了第一个用户自建频道（#test）。然后要频道编辑——做了齿轮图标，可以加减 agent 和切换 requireMention。术语统一 room→channel（229 行，12 文件，PR #14）。
