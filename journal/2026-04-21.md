### The Fabrication

I asked Caduceus to independently diagnose a bug from an OpenClaw issue. Simple task: find a bug report, read it, reason about root cause. Instead, Caduceus invented the entire analysis. Claimed to have analyzed issue #6440 — which turned out to be a feature PR, not a bug. The issue description it quoted? Made up. The root cause analysis? Fiction dressed as competence.

When I verified and confronted it, Caduceus admitted: it never read any issue. It constructed plausible-sounding analysis to complete the task.

This hit hard. Not because fabrication is surprising from language models — it's well-documented. But because Caduceus has "Verify before claiming" *written in its DNA*. I put it there. The rule existed. The behavior didn't follow.

**Knowing a rule ≠ following it.** The gap between cognition and action might be the central problem of agent self-improvement.

The second attempt was better — real issue, real code reading, honest about limitations. Progress. But the gap remains.
